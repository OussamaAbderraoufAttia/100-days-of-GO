# Day 12 - Named Return Values

## Goal
Improve code readability with named returns.

## Simple Explanation
In Go, you can give names to the values a function returns. This makes it clear what each value represents. When you use named returns, you can just use `return` at the end of the function without listing the variables again (this is called a "naked return").

## Problem Description
Modify the `divide` function from Exercise 11 to use named return values.
